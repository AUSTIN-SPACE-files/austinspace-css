The challenge

Starting from nothing

AUSTIN_SPACE launched as a sole trading consultancy with no prior online presence. The Squarespace site had to do everything at once: establish a brand identity, communicate a premium service offering, generate enquiries, and demonstrate the level of craft being sold. All while working within Squarespace's hard platform constraints.

Strategic positioning

  • No existing brand presence or online reputation to draw on
  • Solo consultancy competing directly against full-service agencies
  • Needed to communicate the value of direct access over agency overhead

Technical constraints

  • Squarespace imposes unavoidable overhead: render-blocking resources and legacy JS are baked into the platform
  • Inline <style> blocks stripped from page code blocks, so CSS architecture required careful planning from day one
  • No server-side access; all performance gains had to be achieved within platform limits

Brand & functional requirements

  • Logo, typography, colour system, and tone of voice all needed creating simultaneously
  • Enquiry system needed full analytics tracking, not just a contact form
  • GA4 custom events required to capture form location, service selected, and form type
  • GDPR-compliant data handling and cookie management required from day one
The solution

How it was built

Phase 1

Brand Foundation & Design System

  • Logo designed and implemented across the full site
  • Custom .as-* BEM namespace with CSS custom properties established: --as-purple, --as-off-white, --as-border, --transition and more
  • Typography stack defined: Exo 2 800 for headings, IBM Plex Mono for accents, IBM Plex Sans for body copy
  • Font Awesome Pro (Sharp Thin) integrated throughout
  • Straightforward, confident tone of voice applied consistently across all copy
Phase 2

Build & Technical Foundation

  • 7+ page Squarespace architecture built from scratch with custom HTML in all page code blocks
  • All CSS consolidated into a single master stylesheet in Design → Custom CSS; page code blocks contain HTML only
  • Unified lightbox enquiry form injected globally via Code Injection Footer and triggered from any CTA across the site
  • Formspree integration with fetch() and full client-side error handling
  • GTM installed; GA4 configured with custom as_form_submission events: form_location, service_selected, and form_type parameters registered in GA4 Admin
  • Legal compliance suite built in AUSTIN_SPACE styling: Privacy Policy, Cookie Policy, Terms of Service
Phase 3

Performance & SEO Optimisation

  • Google Fonts preconnect and preload injected via Code Injection header. Removing this was confirmed to worsen LCP from 6.3s to 11.6s in live regression testing
  • Page-level SEO titles and meta descriptions written and applied across all pages
  • Image alt text completed site-wide; policy pages set to noindex
  • CookieYes integrated for GDPR-compliant cookie consent management
Phase 4

AI Search & Ongoing Evolution

  • Google Business Profile established as a service-area business: Worcester and West Midlands, category Web Designer, open Mon to Fri 09:00 to 17:00
  • GBP verified and active
  • JSON-LD structured data implementation complete throughout site
  • AI search optimisation, ensuring the site surfaces across ChatGPT, Perplexity and Gemini results, active and ongoing
  • Every technique and approach is tested on AUSTIN_SPACE before being recommended to clients

FAQ

Common questions

Why did you build your own site on Squarespace?

Because it's the platform I specialise in, and the site needed to be a genuine demonstration of what Squarespace is capable of in the right hands, not a showcase built on a different stack. Every technique applied to client projects is applied here first.

How long did the AUSTIN_SPACE site take to build?

The visual direction came together early, but the site went through multiple iterations before reaching a point worth publishing. Client work takes priority, and the honest answer is that building your own site as a solo operator is one of the harder projects to find time for.

What makes this site technically different from a standard Squarespace build?

Everything visible is built with custom HTML and CSS. No native Squarespace content blocks. The design system, the component library, the forms, the analytics tracking, the structured data, the performance optimisation, all built from scratch within Squarespace. It incorporates everything learned across two years of intensive work on Terracotta Property.

How was AI involved in the AUSTIN_SPACE build?

Claude AI was used as a collaborative tool throughout the Squarespace build, working through CSS architecture decisions, drafting and refining copy, generating schema markup, and building out the FAQ content across every page. It's a working example of how AI can accelerate a professional build without replacing the expertise behind it.

Is this site a fair representation of what a client project looks like?

Yes, and in some ways it's a higher bar. Every client project is built with the same rigour, but this site also had to demonstrate that rigour publicly. It's the most complete example of the full service offering in one place.
