The challenge

Where things stood

Terracotta had outgrown their previous designer. The firm had grown into a respected London consultancy with a strong team and a broad service offering, but the Squarespace site hadn't kept pace. Dated, unoptimised, and failing to reflect the business it was representing. No meaningful SEO. Poor mobile performance. No clear way for the site to work as a business tool.

Technical issues

  • Slow page load times and poor mobile performance
  • Missing meta descriptions and broken heading hierarchy
  • URLs with temporary redirects causing SEO issues
  • Accessibility issues with heading nesting
  • No structured data or schema markup

Functional gaps

  • No cookie policy or privacy-compliant setup
  • No Google Analytics or Search Console integration
  • No Microsoft Clarity for user behaviour tracking
  • No mechanism to attribute enquiries to the website

Design & content issues

  • Outdated design not reflecting the brand
  • Poor copy structure across service pages
  • No clear calls to action or defined conversion paths
The solution

How it was approached

Phase 1

Complete Squarespace Rebuild: 40+ hours

  • Built on Terracotta's existing Squarespace account using a staging environment throughout
  • 40+ pages created from scratch with consistent structure and strong information architecture
  • Full-page design using custom code blocks throughout
  • Mobile-first responsive implementation
  • Custom Font Awesome icons integrated across the site
Phase 2

Technical Foundation & Compliance

  • Cookie & Privacy Policy implementation
  • Cloudflare CDN configured for improved performance
  • Google Analytics & Search Console integration
  • Microsoft Clarity user behaviour tracking
  • Accessibility improvements throughout
Phase 3

Performance & SEO

  • 541 images optimised sitewide, including all blog posts
  • Strategic alt text added to every image using targeted industry keyword research
  • Meta descriptions written and applied across all 65 pages
  • H1 hierarchy and internal linking corrected throughout
  • Blog SEO infrastructure rebuilt: URLs, metadata, breadcrumbs and schema
Phase 4

AI Search & Ongoing Development

  • Comprehensive schema markup: organisation, services, FAQs, reviews and breadcrumbs
  • FAQ sections built specifically for AI search coverage across ChatGPT, Perplexity and Gemini
  • Cloudflare configured to allow AI crawler access
  • Google Ads campaign planned, set up, monitored and reported
  • Looker Studio dashboard built for ongoing performance visibility

FAQ

Common questions

What does a full Squarespace rebuild actually involve?

Every page is designed and built from scratch, not reskinned from a template. That means custom layouts, a considered information architecture, mobile-first development, and all the technical groundwork: SEO, analytics, structured data, and performance optimisation. Everything is done in a staging environment so the live site is never disrupted.

How long did the Terracotta project take?

The initial rebuild was completed in around 40 hours. From there, the relationship moved into an ongoing retainer covering SEO, AI search optimisation, performance work, and continued development, totalling 250+ hours over the course of the partnership.

What does the ongoing partnership element involve?

Once the foundation is in place, the focus shifts to continuous improvement. New pages, SEO refinements, schema updates, performance monitoring, and strategic consultation. The Squarespace site is treated as a living business tool, not a completed project.

How do you measure whether the work is making a difference?

Through a combination of Lighthouse scores, Google Search Console rankings, Analytics traffic data, and conversion signals. For Terracotta, a Looker Studio dashboard was built to give the client a clear, ongoing view of performance without needing to dig into multiple platforms.
